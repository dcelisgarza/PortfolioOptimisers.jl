1 · Forward, Backward, Full, Semi — the four most generic English words, exported bare

1
Files

src/08_Moments/22_StepwiseRegression.jl:63,77,667
src/08_Moments/01_Base_Moments.jl:514,549,962

Problem → Solution

Friction: using PortfolioOptimisers injects Forward/Backward/Full/Semi into the caller's namespace — maximal collision risk, zero domain signal.
Signal: name the algorithm family in the word, so autocomplete and the bare name both carry meaning.

Before — caller sees
export PValue, Forward, Backward
export Full, Semi

# at a call site:
StepwiseRegression(; alg = Forward())
SecondMoment(; alg1 = Semi())

# in the caller's own code:
Forward # ← whose Forward?
After — direction
StepwiseRegression(; alg = ForwardSelection())
StepwiseRegression(; alg = BackwardElimination())

# Full / Semi are documented domain
# terms (CONTEXT.md "Full vs Semi") —
# keep bare, but Forward/Backward
# carry no family signal alone.
Wins
  • Signal: name states the selection method
  • Lower collision risk in caller namespace
  • Fails ADR-0015 test: not "bare when unambiguous"
  • Blast radius: 2 types + exports + docs
ADR 0015 governs RiskMeasure/Estimator suffixes, and its spirit is "bare when unambiguous." Forward/Backward are maximally ambiguous English words — the spirit argues for renaming them, not for keeping them bare.
