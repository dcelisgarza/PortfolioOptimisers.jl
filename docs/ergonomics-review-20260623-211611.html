2026-06-23 · branch dev · friction in caller effort, not implementer effort · relational queries via graphify

Baseline is strong. The error culture already names the mistake and the fix — e.g. "`$(typeof(r))` cannot be evaluated on a precomputed return series … Evaluate it through `expected_risk(r, w, X, fees)` …". 139 exported names, one disciplined glossary (CONTEXT.md). These candidates target the pockets that deviate from that standard, not the standard itself.

1 · Restore the argument name in the 24 generic :val validators

Strong errors
src/01_Base.jl — assert_nonempty / assert_finite / assert_nonneg / assert_gt0 (default sym = :val)
Before — caller omits the symbol (24 of 171 call sites)
assert_nonempty(w) # sym defaults to :val
ERROR: IsEmptyError: val cannot be empty
After — same convention the other 147 call sites already follow
assert_nonempty(w, :w)
ERROR: IsEmptyError: w cannot be empty
Problem. 24 validators fall back to :val, so the caller is told "val cannot be empty" — implementation state, not the argument they passed.
Solution. Pass the real symbol at those 24 sites (the helpers already accept it); the message names the caller's argument.
coverage today
147 sites name the arg
24 say "val"
  • Signal: message names caller's argument
  • Convention: matches the 86% majority
  • Blast radius: 24 call sites, additive
  • No public signature change

2 · Literature acronym aliases (HRP, CVaR, NCO …)

Worth exploring aliases
src/20_Optimisation/, src/19_RiskMeasures/
Before — caller knows the acronym, must discover the spelling
HierarchicalRiskParity()
HierarchicalEqualRiskContribution()
NestedClustered(...)
ConditionalValueatRisk()
MeanAbsoluteDeviation()
After — additive const aliases, autocomplete-discoverable
const HRP = HierarchicalRiskParity
const HERC = HierarchicalEqualRiskContribution
const NCO = NestedClustered
const CVaR = ConditionalValueatRisk
const MAD = MeanAbsoluteDeviation
Problem. The literature names (HRP, HERC, NCO, RRB, FRC, NOC, CVaR, EVaR, RVaR, CDaR, EDaR, MAD) have no aliases — grep "^const HRP" returns nothing — so a caller must already know the long spelling to type it.
Solution. Export const ACRONYM = LongName for the names CONTEXT.md already lists as canonical short forms.
Caution — collisions. RB (RiskBudgeting) vs RRB (RelaxedRiskBudgeting), and FRC (FactorRiskContribution) vs FRC-adjacent risk-budget names must be checked before aliasing. Alias only where the acronym is unambiguous in CONTEXT.md.
  • Discoverability: acronym → autocomplete hit
  • Signal: matches domain literature
  • Blast radius: zero, purely additive
  • Risk: namespace clutter, collisions

3 · Error text names the internal field, not the caller's keyword

Worth exploring errors
src/13_Prior/, src/08_Moments/ (high-order prior assembly)
Before — verbatim
ERROR: IsEmptyError: f_kt cannot be empty
ERROR: IsEmptyError: S2 cannot be empty
ERROR: IsNothingError: pr.rr cannot be nothing when f_kt or f_sk is provided
After — names what the caller supplied
ERROR: IsEmptyError: cokurtosis (`kt`) cannot be empty
ERROR: IsNothingError: this HighOrderPrior needs raw returns;
  pass a prior whose `.rr` is set when supplying `f_kt`/`f_sk`
Problem. Internal field names (f_kt, S2, D2, L2, pr.rr) leak into the message; the caller can't map them to the keyword they passed.
Solution. Phrase these few messages in the caller's vocabulary — the same fix-oriented style the precomputed-returns error already uses.
  • Signal: error in caller's vocabulary
  • Convention: matches best-in-repo errors
  • Scope: a handful of high-order-prior sites
  • No behavioural change

5 · port_opt_view is hard to guess — surface it where callers meet it

Speculative discoverability
src/13_Prior/01_Base_Prior.jl + 9 more — marked public, not exported

Problem. Said aloud, "port opt view" fails the signal test — a caller can't guess it means "sub-select a portfolio estimator to an index". It is the only public (1 of 1) non-exported entry point, so it must be discovered by reading source.

Direction. Keep the name; raise its discoverability — a docstring cross-link from factory ("the value-propagation counterpart of port_opt_view"), and a CONTEXT.md "View" → entry-point pointer. The asymmetry with factory (short verb vs three-token name) is the friction; an exported short alias such as subselect is the speculative end of this.

ADR 0010 deliberately chose port_opt_view when unifying the view family (18 references). The name is settled — this candidate is about discoverability around it, not a rename. Do not re-open the naming decision.

Respected as settled — not re-suggested

  • ADR 0009 — matrix-processing order step symbols (:pdm :dn :dt :alg) are a decided convention, not friction.
  • ADR 0015 — disambiguation suffixes (FullMoment / SemiMoment, ForwardSelection / BackwardElimination) are settled naming.
  • ADR 0005 — prefix-namespaced risk state is an intentional convention.
