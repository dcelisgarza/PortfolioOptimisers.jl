Two adapters justify the seam: returns feed CVaR, drawdowns feed CDaR — one cone, tested directly on a returns vector.
Problem
The cone formulations — the mathematically interesting half of every risk measure — are reachable only through a half-assembled JuMP model, and the returns/drawdown twins (CVaR↔CDaR, EVaR↔EDaR, RVaR↔RLDaR, PNVaR↔PNDaR) are hand-mirrored despite the analytic functors already sharing one kernel.
Solution
Extract each cone as a pure builder over a returns/drawdown expression (cvar_cone(expr, α)); the emitter's job shrinks to choosing which series to feed it.
  • ▸ interface is the test surface: cone tested on a vector
  • ▸ leverage: one cone, N feeders
  • ▸ twins collapse: CVaR/CDaR share one builder
  • ▸ locality: a stabilisation change lands once, not ×4

One seam absorbs both keying schemes; the ordering invariant becomes a property the interface states, not a comment on SHARED_STATE.

Problem
Builder correctness depends on ordering and hidden presence flags that appear in no signature; the MIP layer uses a parallel keying scheme outside the fail-closed lock.
Solution
Have each builder declare the Model State it reads and writes; derive the assembly order (or assert it) from those declarations, and pull mip_key under the same interface.
  • ▸ ordering invariant becomes checkable
  • ▸ one Model-State vocabulary, not two
  • ▸ seam-lock actually covers the MIP layer
  • ▸ locality: a reorder fails fast, not silently
Extends ADR 0037 rather than contradicting it — 0037's own "still out of scope" names :noc_rk, :psi, :w_obj as unclassified. This is the temporal contract it deferred.

After — one combinator dispatches on a typed presence wrapper
optional(builder, input)

Enabled/Disabled is a type. A mis-shaped call can no longer match a no-op.
Problem

Solution
A single optional-builder combinator that dispatches on an enabled/disabled presence type; the disabled branch is the only path that skips.
  • ▸ delete ~22 shallow no-op wrappers
  • ▸ mis-shaped calls raise, not vanish
  • ▸ locality: the skip rule lives in one module
  • ▸ finishes what ADR 0036 started for hooks

Problem

  • ▸ interface shrinks; math concentrates

  • ▸ new algorithm = one method
  • ▸ cov2cor lives once, not ×8

To pay off, it must stamp families — the Range/Drawdown/Relative/DR axis (a data-carrying kind parameter, as LowOrderMoment{…,alg} already does), not single measures.

Problem
The 48 measures are a hand-maintained cross-product; risk_input_kind is an implicit reflection contract you must remember to extend, and the DR twins even carry validated fields (l, r) that the analytic functor silently ignores — a reporting/optimisation mismatch.
Solution
A macro that emits the struct, constructors, docstring skeleton, and the input-kind/precomputed contract from one declaration keyed on the family axis.
  • ▸ contract can't drift out of sync
  • ▸ the Range/DR axis becomes data, not N structs
  • ▸ best done after #01 gives the emitter a seam
  • ▸ surfaces the inert l/r mismatch
Already tracked (ADR 0018 "M5"; memory notes it as future work). Listed for completeness and re-scoped: it should stamp families, and it depends on #01 for the emitter half.
