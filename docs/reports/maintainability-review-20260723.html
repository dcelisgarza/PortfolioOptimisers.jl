The ρp correlation leak is gone; the wider dist_transform consolidation (and the uneven square-guard) remains open.

Resolved (2026-07-23). A generic Statistics.cov(ce::AbstractCovarianceEstimator, X; …) = cor2cov!(cor(ce, X; …), std(ce.ve, X; …)) fallback now lives in 01_Base_Moments.jl, beside the existing compat_cov/robust_cov layer and symmetric to the var/std-from-cov derivations. Top recommendation — extract weighted_quantile and fix the drift it exposes

Status (2026-07-23): the three latent bugs (cards 1, 2, 5) are fixed on dev. What remains of this recommendation is the consolidation — the weighted_quantile helper and the card-3 emit_xatrisk! skeleton — which would prevent this class of drift from recurring.

Start with card 1. It is the highest-leverage item because it is both a live correctness bug and a clean consolidation: five of six weighted-quantile sites scale the search target by sw and drawdown_at_risk does not, so weighted DaR/RLDaR silently return a wrong quantile for un-normalised weights. A single weighted_quantile(v, level, w) helper collapses the six sites to one-liners and forces the diverged copy back onto the family convention — the fix and the consolidation are the same edit. Pair it with card 2 (the DR-CDaR key collision), the second latent bug, since both are copy-paste residue of the twin structure that card 3 would eliminate at the source. After the bugs, card 4 (Fees, ~34 sites) and card 5 (Distance, ~48 methods) buy back the most blast radius.
