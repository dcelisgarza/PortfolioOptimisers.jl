Problem: the rule “which Pipeline Context slot lands in which optimiser field” lives in the Pipeline as string-keyed PropertyLens writes — the Pipeline’s interface to the optimiser is the optimiser’s private field layout, and a field rename breaks the Pipeline at runtime, far from the rename.

Solution: each Optimisation Estimator owns one method — apply_context_slots(opt, ctx) — and the Pipeline calls it blind; the slot→field map moves behind the optimiser’s interface.

  • Locality: field knowledge lives with the fields
  • Seam becomes real: two adapters (JuMP, Hierarchical)
  • Routing unit-testable per optimiser, solver-free
  • Duplicated WeightBounds/constraint loop collapses
ADR 0028 defines the Pipeline seam but not who owns the routing — no conflict, this sharpens it.

3 · Model State: typed accessors instead of the regex seam-lock

Worth exploring

src/20_Optimisation/08_Base_JuMPOptimisation.jl (accessors) · test/test_28_seam_lock.jl (the regex) · ~22 emitter files, e.g. 20_RiskMeasureConstraints/06_XatRiskConstraints.jl:147

Before — convention policed after the fact

model[Symbol(:var_risk_l_, i)] — hand-built keys
accessor layer (bypassable)
test_28 regex lint — admits its own hole: a sloppy Symbol(prefix, :newCatA) slips through

After — bare reads impossible to write

Model State module

risk_var(model, i; prefix)

key construction, prefix discipline,
Cat-A/Cat-B routing — all internal

the invariant moves from lint to interface

Problem: the Model State interface is a Dict{Symbol,Any} plus a naming convention every emitter must know — the invariant “never read a prefix-managed key bare” is enforced by a regex over source text, which is a seam held shut by lint, not by types.

Solution: widen the accessor family until emitters never build a key by hand; the prefix and category discipline become implementation.

  • Interface shrinks: emitters learn accessors, not conventions
  • The seam-lock test becomes redundant, then deletable
  • Locality: key policy changes in one module
ADRs 0004/0005/0033/0034 already moved this direction and knowingly accepted the residual hole — reopen only as incremental accessor-widening, not a rewrite.
