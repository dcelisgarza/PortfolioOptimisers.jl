Architecture review — PortfolioOptimisers.jl

2026-07-07 · branch dev

This pass hunted for shallow modules in the risk-measure layer — the one part of the pipeline the last review left open. The headline: the library already contains its own deep exemplar (GenericValueatRiskRange, a 12-line delegation over the loss seam), yet six concrete Range measures still hand-duplicate the formulation it replaces, and three tail-risk families instantiate the same four-member template around interchangeable cone kernels. The JuMP optimiser spine (post-ADR-0008) and the plotting extension seam checked out as genuinely deep — no action there.

Top recommendation — collapse the Range twins onto GenericValueatRiskRange, harness first

Start with #3 and #1 as one move: extend the ADR-0008 solver-free harness to assert each Range measure's Model State keys, then collapse the six hand-duplicated _l/_h formulations onto the 12-line GenericValueatRiskRange delegation that already ships. The deep module, the seam (loss::Bool), and the exemplar all exist — this is deletion guarded by a new test surface, not new design. It also derisks #2: once the Range members of each tail family are delegations, the Entropic/Relativistic/PowerNorm template shrinks from four members to three, making the kernel-unification a smaller, safer second step.
