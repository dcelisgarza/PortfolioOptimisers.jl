1 · Complete the Model State seam

Strong in-process

src/20_Optimisation/08_Base_JuMPOptimisation.jl · 09_JuMPConstraints/* · 14_RiskBudgeting.jl · 15_RelaxedRiskBudgeting.jl · 20_RiskMeasureConstraints/01_BaseRiskConstraints.jl

Before — partial seam

Named layer (ADR 0004) covers the hot keys; ~40 tail keys read raw straight past it.

After — every multi-reader key through the seam

Builders never touch model[:sym]. One place declares what each key holds and whether it is set.

Problem. The seam is half-built: sw/lw (10 reads each), risk_vec/risk_frontier/fees (4), w1/psi/ohf (3) are pulled raw across separate builder files with no getter, no set-before-read check, no type contract.

Solution. Extend the ADR 0004 accessor layer to every key read by two or more builders — the same get_*/has_* shape already proven for get_w, get_risk, get_dd.

  • leverage: builders read one named interface
  • locality: key meaning lives in one place
  • two adapters: each key has ≥2 readers — real seam
  • read-before-write becomes a defined error, not a wrong slice
  • interface is the test surface for assembly order
  • tail keys stop leaking past the seam
Aligns with ADR 0004 — which already says "prefer these over raw model[:sym]." This completes that decision rather than reopening it. Stop at multi-reader keys; a getter for a once-read key is a hypothetical seam, not a real one.

3 · Unify the XatRisk Range combinator

Worth exploring in-process

06_XatRisk.jl · 07_ConditionalXatRisk.jl · 08_EntropicXatRisk.jl · 09_RelativisticXatRisk.jl · 19_PowerNormXatRisk.jl

Before — every Range re-hand-codes lower + upper tail

EntropicValueatRiskRange(x) =
  ERM(x, slv, alpha, w) + ERM(-x, slv, beta, w)

RelativisticValueatRiskRange(x) =
  RRM(x, slv, alpha, kappa_a, w)
    + RRM(-x, slv, beta, kappa_b, w)

PowerNormValueatRiskRange(x) =
  PRM(x, slv, alpha, pa, w)
    + PRM(-x, slv, beta, pb, w)

5 families, same shape: m(x, lower) + m(−x, upper).

After — one combinator over a base measure

struct Range{R}     # deep module
  lower::R          # base measure, lower tail
  upper::R          # base measure, upper tail
end

(r::Range)(x) = r.lower(x) + r.upper(-x)
VaR CVaR EVaR RVaR PNVaR → one Range

Problem. Each of the 5 Range variants is a hand-written struct + constructor + functor pair that re-expresses the identical "base on +x, base on −x, add" composition.

Solution. One Range combinator parameterised by a base measure for each tail; the functor is the shared one-liner.

  • leverage: one combinator, N base measures
  • locality: the +x/−x idea lives once
  • delete 5 shallow functor pairs
  • new tail measure gets Range for free
Caveat (memory: arch-review item 4). The analytical functor unifies cleanly; the JuMP formulation in 20_RiskMeasureConstraints differs per family (cones differ). Unify the net-returns side first; treat set_risk_constraints! as a separate, per-family pass.

5 · Fold the five Windowed wrappers into one

Worth exploring in-process

src/08_Moments/28_WindowedExpectedReturns.jl · 29_WindowedCovariance.jl · 30_WindowedVariance.jl · 31_WindowedCoskewness.jl · 32_WindowedCokurtosis.jl

Before — 5 wrappers, same body

Each: {est, w, window} + one method delegating to windowed_preamble. ~20 lines of wrapper for 3 lines of body.

After — one parametric wrapper

One struct; the inner estimator's own method dispatches the windowed call.

Problem. Five wrappers carry identical fields and a 3-line body around a shared windowed_preamble; the interface is as wide as the implementation — shallow.

Solution. One Windowed{E} parameterised by the inner estimator; the windowed method delegates through the inner estimator's own moment call.

  • interface shrinks; one struct absorbs five
  • locality: window logic lives once
  • delete ~100 lines of wrapper
  • new windowed moment for free
Check first. The five delegate to differently-named moment entry points (cov, estimate_expected_returns, coskewness…) and subtype different estimator supertypes. A single Windowed{E} can't subtype all of them at once — confirm dispatch resolves (trait on the inner estimator, or a small method per entry point) before collapsing. If the per-supertype methods can't be avoided, the win shrinks to merging the struct only.
